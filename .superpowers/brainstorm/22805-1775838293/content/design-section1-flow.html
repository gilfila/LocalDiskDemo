Design — Section 1: Page Flow & Navigation

How users move through the app

Worker Flow
📄 home.pmd Route: /
Shows worker profile + registration status card
• No car registered → "Register My Car" button
• Pending approval → status badge
• Approved → car details + OTP history
↓ clicks Register
📄 register-car.pmd Route: /register
Step 1 — Car Details
Make · Model · Year · License Plate · State
Auto-checks home address distance on load

Step 2 — Review
Summary of entered details + distance result
Commuter benefit eligibility shown

Step 3 — Confirmation
"Submitted for manager approval" screen
↓ submit → manager approval BP
Workday Business Process routes to manager inbox
On approve → registration active, OTP anniversary set
On deny → worker notified, can re-register
Manager Flow
📄 home.pmd Route: /
Same home page — shows manager's own registration
+ "Manage My Team's Commuters" button if manager role
↓ clicks Manage Team
📄 manage-commuters.pmd Route: /manage
Table of direct reports with registered cars
Columns: Name · Car · Distance · Status · Last OTP · Actions

Actions per row:
• Approve / Deny pending registrations
• Trigger Ad-hoc OTP ($100)
• View registration details
Scheduled OTP (background)
Orchestration runs nightly · checks anniversary dates · auto-triggers $100 OTP for eligible approved workers
Pages summary (PMD files)
• home.pmd — worker status hub
• register-car.pmd — 3-step wizard
• manage-commuters.pmd — manager table
