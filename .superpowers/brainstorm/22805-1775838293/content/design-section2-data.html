What we store and what runs behind the scenes

Custom Business Object — CarRegistration
workerIdText — Workday worker ID (key) makeText modelText yearNumeric licensePlateText plateStateText statusText — pending / approved / denied distanceMilesNumeric — calculated at registration isEligibleBoolean — distanceMiles > 20 registrationDateDate approvedDateDate lastOtpDateDate — last OTP issued otpCountNumeric — total OTPs issued
Orchestrations (6)
getWorkerEligibility
Fetches worker home address + nearest office location → calculates distance via Haversine formula → returns miles + eligibility flag
submitCarRegistration
Saves CarRegistration object with status=pending → triggers manager approval business process
getWorkerRegistration
Returns current worker's CarRegistration record for home page status display
getTeamRegistrations
Returns all CarRegistration records for manager's direct reports — powers the manage-commuters table
issueOtp
Issues $100 OTP payment for a worker → updates lastOtpDate + otpCount on CarRegistration. Used by both scheduled and ad-hoc flows.
anniversaryOtpScheduler
Scheduled nightly — queries all approved+eligible registrations → filters where today = anniversary of registrationDate → calls issueOtp for each match
